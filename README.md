# Forager

🇮🇹 **[Leggi in italiano](README.it.md)** · 🌐 [forager.michelangelogigli.it](https://forager.michelangelogigli.it) · ⬇️ [Download for Mac (.dmg, notarized)](https://forager.michelangelogigli.it/downloads/Forager-1.1.0.dmg?v=4)

**The open-source CRM for major donor, corporate and foundation fundraising.** Not a generic
CRM: it's built for people who raise money from major donors, companies and foundations.
Everything runs locally — no cloud, no subscriptions (except the AI engine, see [Costs](#costs)).

Type a name → Claude runs public web research, synthesizes a structured profile (capacity,
propensity, board seats, giving history, network), fills the CRM with verifiable sources and
suggests the next best action aligned with your causes and ask tiers. Then it stays with you
through the whole cycle: follow-up cadences, a real gift register, campaigns, thank-yous, forecast.

📖 **[Complete guide](docs/GUIDE.en.md)** ([Italiano](docs/GUIDA.md)) · the same guide ships in-app.

> **Note on language**: the app UI is fully bilingual (English/Italian — switch anytime from the
> sidebar). Some compliance features (deductibility flags, receipts) follow Italian nonprofit rules.

## What it does

**Research & intelligence**
- **AI prospect research** — profile generated by Claude from public sources, with citations
- **Source verification + grounding** — checks that sources exist AND actually support the claims
- **Deep dives** per section (wealth, board, giving), **news with signals** (opportunity/risk)
- **Ask your data** — natural-language chat over the whole database
- **Network mapping** — relationship graph across prospects, boards and foundations

**A real CRM**
- **6-stage pipeline** with probability-weighted forecast
- **Gift register** (received vs promised) with deductibility and receipts (Italian compliance)
- **Campaigns** with targets and attributed revenue
- **Follow-up cadences** — a "reach out today" queue on the dashboard
- **Thank-you cycle** — a "to thank" queue after every gift
- **Trash** — deleted prospects can be restored
- **Tasks, tags, goals, activity feed, CSV import/export + full JSON dump**

**Writing**
- **AI Compose** — email drafts grounded in the prospect + your organization
- **Multi-step sequences**, email examples, reusable snippets

## Install

### Requirements

- Python 3.10+
- One AI engine of your choice, installed and authenticated:
  - [Claude Code CLI](https://docs.claude.com/claude-code) by Anthropic (recommended), **or**
  - [Codex CLI](https://developers.openai.com/codex/cli) by OpenAI (select it in Settings or with `FORAGER_AI_ENGINE=codex`)
- macOS / Linux / Windows
- (optional) A free Hunter.io account for decision-maker email lookup

### macOS app (no terminal)

Download the **[notarized .dmg](https://forager.michelangelogigli.it/downloads/Forager-1.1.0.dmg?v=4)**,
drag Forager into Applications, double-click. Apple Silicon only.

### One-liner (macOS / Linux)

```bash
curl -fsSL https://raw.githubusercontent.com/Mic-Fundraiser/forager/main/install.sh | bash
```

### Manual install

```bash
git clone https://github.com/Mic-Fundraiser/forager.git
cd forager
./forager init
./forager start
```

Your browser opens automatically at <http://127.0.0.1:5000>.

### Windows

Download the ready-made **[zip](https://forager.michelangelogigli.it/)** (extract, double-click `Forager.exe`), or from source:

```cmd
git clone https://github.com/Mic-Fundraiser/forager.git
cd forager
forager.bat init
forager.bat start
```

### Docker (alternative)

```bash
docker compose up -d
```

> **Note**: inside Docker the AI engines (claude/codex) are unavailable (they need the authenticated
> host binary). For AI research, run Forager natively with `./forager start`.
> The port binds to `127.0.0.1` only because the app has no authentication: for remote access
> use an authenticated reverse proxy or a VPN.

## Costs

Forager is free (MIT). The AI engine is a paid CLI you invoke locally with YOUR subscription:
**Claude Code** (Anthropic, recommended) or **Codex CLI** (OpenAI, included in ChatGPT Plus/Pro) —
no key ever passes through us, no cost beyond what you already pay your provider. Every AI action
is explicit (no background calls) and the **AI Usage** page shows what you consume (with Codex the
CLI doesn't report dollar cost: you still see tokens and duration). Hunter.io is optional with a
free tier (25 searches/month).

## Configuration

Everything is optional — Forager works out of the box. To customize, copy `.env.example` to `.env`:

```bash
cp .env.example .env
```

Main variables:

| Variable | Default | What it does |
|---|---|---|
| `FORAGER_PORT` | `5000` | Web server port |
| `FORAGER_HOST` | `127.0.0.1` | `0.0.0.0` for LAN access (discouraged: no auth) |
| `FORAGER_SECRET_KEY` | auto | Generated and stored in `data/.secret_key` on first run |
| `FORAGER_AI_ENGINE` | `claude` | AI engine: `claude` (Claude Code) or `codex` (OpenAI Codex CLI) |
| `HUNTER_API_KEY` | empty | Your [hunter.io](https://hunter.io) key (free tier OK) |
| `HUNTER_SENIORITY` | `executive` | Decision-maker filter (`executive`/`senior`/`junior`) |
| `CLAUDE_BIN` | auto | Path to the `claude` binary if not in PATH |
| `CODEX_BIN` | auto | Path to the `codex` binary if not in PATH |
| `CODEX_MODEL` | CLI default | Codex model (e.g. `gpt-5-codex`) |

## First run

1. Open <http://127.0.0.1:5000> — on first launch you get the **welcome wizard**: org name, mission, causes, typical asks (60 sec)
2. Go to **Prospect Research** → type a prospect's name → the full profile is ready in 1–3 minutes
3. Open the profile → set the stage, a **follow-up cadence**, log **gifts**, draft emails with AI Compose
4. Create **campaigns** and link gifts to them: the dashboard tells you what's working
5. Refine your briefing under **My org** (more detail = sharper research)

## Troubleshooting

**"The AI engine is unavailable"** (yellow banner on the dashboard)

With the **Claude Code** engine (default):
```bash
npm install -g @anthropic-ai/claude-code
claude   # first run: log in with your Anthropic account
```
With the **Codex** engine (if selected in Settings):
```bash
npm install -g @openai/codex
codex login   # log in with your ChatGPT account
```
Then restart Forager. If the binary is installed but not in PATH, set `CLAUDE_BIN=/path/to/claude`
(or `CODEX_BIN=/path/to/codex`) in `.env` or from Settings.

**PDF export doesn't work on macOS** — WeasyPrint needs system libraries:
```bash
brew install pango libffi
```
Alternatively use "Print → Save as PDF" from the browser (Print button on the profile), which needs nothing.

**Hunter says "Error"** — open Settings: the message tells you whether the key is missing, expired or over quota.

**"Database is locked"** — close duplicate Forager instances (`./forager start` twice). WAL and retries are already enabled.

**Port 5000 is busy** (on macOS it's often AirPlay) — set `FORAGER_PORT=5001` in `.env`.

**Full diagnostics**: `./forager doctor`

## Privacy & data

- **Everything local**: SQLite database in `data/crm.db`. No cloud, no telemetry.
- **Your data is yours**: CSV exports (prospects, gifts) and a **full JSON dump** from Settings. Zero lock-in.
- **Automatic backups** every 24h into `backups/` (14 kept) + manual `./forager backup`.
- **AI research**: prompts go through the local AI CLI (your own subscription).
- **Hunter**: if configured, it only ever receives the company domains you search.
- **Avatars and logos**: generated locally (SVG) — no third-party calls.
- **Offline frontend**: all libraries are served locally (`static/vendor/`), the UI works without internet.

## Ethics

- **Public sources only**: press, business registers, institutional bios, official statements, public LinkedIn, foundation records.
- **GDPR-aware**: profile only people with public exposure, for legitimate-interest purposes (transparent fundraising).
- **Estimated capacity = an estimate**: always verify before a formal ask.
- **Exclusion criteria**: define them in "My org" → Claude flags red flags when a prospect hits your no-go areas (industries, conflicts of interest).

---

## Stack

- **Backend**: Python 3, Flask, SQLite (no heavy external dependencies)
- **AI engine**: Claude Code CLI or OpenAI Codex CLI as a subprocess (your subscription, no API keys)
- **Frontend**: Tailwind CSS, Lucide icons, HTMX, Alpine.js — all served locally, zero build step
- **Email search**: Hunter.io API (optional)
- **Security**: CSRF on all POSTs, auto-generated secret key, localhost-only bind, non-root Docker
